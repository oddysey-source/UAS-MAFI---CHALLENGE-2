Panduan Aplikasi

  1. Halaman Awal Aplikasi
    Saat aplikasi dibuka, pengguna akan diarahkan ke halaman About Me yang berisi informasi pembuat aplikasi dan deskripsi singkat project UAS. Navigasi menu tersedia di bagian atas untuk berpindah halaman.
  2. Menu Input Anggota
    Menu ini digunakan untuk melihat data awal anggota keluarga dan menambahkan anggota keluarga baru ke dalam struktur tree.

    Langkah menambahkan anggota keluarga:
    • Masukkan Nama Anggota.
    • Masukkan Peran (Ayah, Ibu, Paman, Anak, dll).
    • Masukkan Nama Orang Tua sesuai dengan node yang sudah ada.
    • Upload foto anggota dengan ketentuan:
      • Format file .jpg
      • Ukuran maksimal 100 KB
      • Upload bersifat wajib
    • Klik tombol Tambah Node.
    Jika parent ditemukan, node baru akan langsung ditambahkan ke dalam tree.
  3. Menu Visualisasi Tree
    Menu ini menampilkan struktur silsilah keluarga dalam bentuk pohon (tree). Informasi yang ditampilkan:
    • Jumlah Node: total anggota keluarga saat ini.
    • Kedalaman Tree: tingkat kedalaman maksimum tree.
    Perbedaan warna node:
    • Abu-abu : Root visual (Nabi Muhammad Ô∑∫)
    • Biru : Node parent (memiliki anak)
    • Hijau : Node leaf (tidak memiliki anak)
    Setiap node dilengkapi dengan:
    • Tombol Edit (‚úèÔ∏è) untuk mengubah nama dan peran.
    • Tombol Hapus (üóëÔ∏è) untuk menghapus node beserta seluruh turunannya, kecuali root data dan root visual.
  4. Penyimpanan Data
    Seluruh perubahan data (tambah, edit, hapus node) akan disimpan secara otomatis ke dalam Local Storage browser. Data akan tetap tersimpan meskipun halaman di-refresh.
  5. Menu Algoritma Program
    Menu ini berisi pseudocode dan penjelasan algoritma yang digunakan, seperti traversal tree, penambahan node, penghitungan jumlah node, penghitungan kedalaman tree, serta penghapusan node.
  6. Menu Struktur Data
    Menu ini menampilkan representasi struktur tree dalam bentuk objek JSON yang menggambarkan relasi parent‚Äìchild pada silsilah keluarga.
  7. Menu Konsep Matematika Informatika
    Menu ini menjelaskan konsep-konsep Matematika Informatika yang digunakan, seperti struktur data tree, relasi parent‚Äìchild, depth, traversal, serta kompleksitas waktu algoritma.
  8. Menu About Me
    Menu ini berisi informasi mahasiswa, mata kuliah, dosen pengampu, dan tujuan pembuatan aplikasi sebagai project UAS.

Algoritma Pseudocode:

Load familyTree from LocalStorage
IF familyTree == NULL THEN
  Initialize default familyTree

FUNCTION renderTree(node):
  Display node
  FOR EACH child IN node.children DO
    renderTree(child)
  END FOR
END FUNCTION

FUNCTION addNode(currentNode, parentName, newNode):
  IF currentNode.name == parentName THEN
    currentNode.children.push(newNode)
    RETURN true
  END IF

  FOR EACH child IN currentNode.children DO
    IF addNode(child, parentName, newNode) THEN
      RETURN true
    END IF
  END FOR
  RETURN false
END FUNCTION

Konsep Matematika Informatika

  • Tree (struktur pohon)

    • Aplikasi ini menerapkan konsep struktur data tree, di mana setiap node merepresentasikan satu anggota keluarga.

  • Relasi parent‚Äìchild

    • Relasi parent‚Äìchild digunakan untuk membangun hierarki silsilah keluarga dalam bentuk tree murni, di mana setiap node hanya memiliki satu parent untuk menjaga kesederhanaan struktur data.

  • Depth (kedalaman node)

    • Konsep depth dan height digunakan dalam aplikasi ini untuk menghitung kedalaman node serta kedalaman maksimum tree. Selain itu, jumlah node ihitung secara rekursif untuk mengetahui kompleksitas struktur data.

  • Traversal konseptual

    • Traversal konseptual diterapkan dalam aplikasi ini ketika sistem menelusuri setiap node pada struktur tree untuk berbagai keperluan, seperti menghitung jumlah node, menentukan kedalaman tree, serta mencari, mengedit, dan menghapus anggota keluarga.

  • Keterbatasan Model Tree

    • Struktur data tree yang digunakan dalam aplikasi ini merupakan tree murni (single-parent tree). Konsekuensinya, setiap node hanya memiliki satu parent. Relasi pasangan (suami‚Äìistri) tidak direpresentasikan secara horizontal, melainkan dimasukkan sebagai child untuk menjaga konsistensi algoritma traversal dan kompatibilitas dengan visualisasi tree.

  • Alasan Pemilihan Struktur Tree

    • Struktur tree dipilih karena mudah dipahami, mendukung traversal rekursif, dan sesuai dengan konsep dasar struktur data yang dipelajari dalam Matematika Informatika. Struktur lain seperti graph atau DAG tidak digunakan karena akan meningkatkan kompleksitas algoritma dan visualisasi.
